# 页面体验改进清单

本文档从用户视角整理当前前端页面的体验改进方向。它是产品和交互改进文档，不代表所有条目已实现。

## 1. 全局体验

### G1：页面高度和滚动策略

现状：列表页已开始使用占满视口的卡片布局，但不同页面之间仍容易出现外层滚动、内层滚动、分页遮挡和少量数据也显示滚动条的问题。

改进：

- 建立统一页面骨架：页头固定、内容区占满剩余高度、卡片内部负责滚动。
- 表格少量数据时不启用内部纵向滚动，多数据时再固定表头和滚动区域。
- 分页固定在表格卡片底部，不覆盖最后一行数据。
- 每个页面只保留一个主要滚动容器，避免外层浏览器滚动条和内层表格滚动条同时出现。
- 设置页、研究页这类表单或图表页允许自然滚动，但不要和表格页混用同一高度策略。

优先级：P0。

### G2：页头连接状态

现状：页头已有页面级 SSE 呼吸灯，但用户需要知道它代表什么连接、是否新鲜、多久未更新。

改进：

- 呼吸灯 hover 展示当前页面 channel、连接状态、最近收到数据时间和失败原因。
- 断线时从呼吸灯进入连接诊断抽屉，展示重试次数和最后错误。
- 研究页如果不使用 SSE，呼吸灯显示“按筛选条件请求”或不显示页面推送状态，避免误解。

优先级：P1。

### G3：数据新鲜度

现状：多个页面展示的是实时快照、历史分析或聚合结果，但页面上不总是清楚显示数据时间。

改进：

- 图表和表格标题区统一显示“最新数据时间”。
- 历史分析页显示数据范围、聚合方式、样本数和最后更新时间。
- 账户、仓位、对冲组显示快照时间和来源。
- 后端返回空数据时区分“尚无数据”“筛选条件无匹配”“接口失败”“权限阻塞”。

优先级：P1。

### G4：危险操作确认

现状：平仓、接管、实盘开关等动作已经有保护，但可以进一步统一交互。

改进：

- 高风险按钮统一使用确认抽屉，而不是简单确认框。
- 确认抽屉展示 readiness、风控、行情同步、账户、仓位和滑点摘要。
- live 或 paper-live 动作需要输入确认短语或二次管理员确认。
- 所有危险操作完成后给出可点击的审计记录和执行记录链接。

优先级：P0。

## 2. 登录页

现状：登录页已避免预填默认密码。

改进：

- 登录失败时区分账号密码错误、服务不可达、系统安全配置阻塞。
- 生产环境默认密码阻塞时给出明确运维提示，但不泄露敏感配置。
- 登录后如果原本访问某个业务页，应回到原页面。

优先级：P2。

## 3. 仪表盘

现状：仪表盘展示摘要卡片、权益曲线和账户表。权益曲线已调整为总权益口径。

改进：

- 权益曲线提供“总权益 / HL / MT5”切换，不同线条清晰区分。
- 曲线 tooltip 展示平台拆分、数据来源和快照时间。
- 账户表和摘要卡片口径保持一致，避免总权益和账户拆分看起来对不上。
- 摘要卡片支持点击跳转到对应页面，例如未读告警跳日志，持仓对冲组跳对冲组。
- 增加系统健康摘要，包括行情、扫描、数据库、MT5、Hyperliquid、SSE。
- 增加 Agent 最新建议入口，显示高风险建议数量和最近诊断时间。

优先级：P1。

## 4. 价差研究

现状：价差研究按筛选条件请求分析接口，不使用 SSE。

改进：

- 页面顶部明确显示“非实时推送，筛选变化后刷新分析结果”。
- 展示样本数、时间范围、聚合桶、最后更新时间和当前入场/退出线。
- 图表支持入场价差、平仓价差、mid 价差切换，并用一致颜色表示方向。
- 缺数据时提示可能原因：品种未启用、扫描未运行、时间范围太短、数据库未聚合。
- 与链路监控联动，当前品种报价异常时给出跳转。
- 后续 Agent 可在本页解释“当前价差是否接近入场/平仓条件”，但不直接展示下单按钮。

优先级：P1。

## 5. 资金费研究

现状：资金费研究按筛选条件请求分析接口，图表横坐标和图例已出现过可读性问题。

改进：

- 横坐标根据 `raw/hour/day` 自动选择时间格式，避免全是同一个时间。
- 图例明确区分“单期资金费率”“累计资金费率”“均值”。
- 资金费正负颜色和说明要统一，清楚表达对哪一边有利。
- 统计卡片增加样本区间、缺失率和最大连续缺失。
- 当前品种没有资金费数据时，提示是否为非永续合约、HIP-3 映射缺失或接口失败。
- 后续 Agent 可提示资金费是否正在侵蚀持仓收益。

优先级：P1。

## 6. 报价时差

现状：报价时差使用页面级 SSE，适合做实时判断。

改进：

- 明确展示 HL 领先、MT5 领先、无明显领先三种状态。
- 图表或列表中展示最大滞后毫秒、跟随比例、最大 mid 差和样本量。
- 当报价不同步或样本不足时，不显示误导性的结论。
- 支持从异常点跳转到链路监控对应品种。
- Agent 可在本页解释“该品种当前是否适合依赖领先信号”。

优先级：P2。

## 7. 链路监控

现状：链路监控承担行情、扫描、候选、对冲池和运行状态诊断。

改进：

- 把每个品种的主阻塞原因放到卡片首屏，减少用户找原因的成本。
- 对阻塞原因增加“下一步建议”，例如检查 MT5 会话、确认品种映射、等待报价恢复。
- 展示后台任务最近运行时间、耗时、失败原因和下一次调度时间。
- 对冲池状态提供按状态筛选和异常状态高亮。
- 增加“复制诊断快照”按钮，方便排查。
- Agent 可复用链路监控证据生成运行诊断。

优先级：P0。

## 8. 对冲组

现状：对冲组是执行生命周期的核心页面。

改进：

- 对冲组列表突出当前状态、方向、开仓价差、平仓价差、入场线、退出线和持有时间。
- 异常状态如 `open_partial`、`closing`、`manual_intervention` 需要明显标记。
- 行展开后展示双腿订单、成交、费用、资金费、隔夜费、当前可平仓估算。
- 平仓、接管、恢复等操作统一进入危险操作确认抽屉。
- 支持按状态、品种、执行模式、异常类型筛选。
- Agent 建议标签直接挂在对冲组行上。

优先级：P0。

## 9. 执行记录

现状：执行记录包含订单和成交，分页和滚动曾出现遮挡问题。

改进：

- 订单和成交使用稳定的上下或标签页布局，分页不遮挡表格。
- pending、failed、partial filled 等状态高亮。
- 每个订单展示 external order id、平台错误、最近回查时间和 reconciler 处理结果。
- 从订单可跳转到对冲组、成交、日志和审计。
- 支持只看异常订单。
- Agent 对 pending 长时间未恢复的订单生成解释和处理建议。

优先级：P0。

## 10. 仓位

现状：仓位页已调整为占满布局，但少量数据也曾出现内部滚动条。

改进：

- 少量数据不显示内部滚动条，空数据仍保持页面稳定占满。
- 明确区分系统对冲组仓位、已接管仓位、外部孤儿仓位。
- 单腿、数量不平衡、方向异常和 closed 残仓要高亮。
- 每行操作区提供“查看关联对冲组”“接管”“建议处理”的稳定入口。
- 仓位数量展示同时包含平台原始单位和内部换算数量。
- Agent 在本页优先展示补残腿和接管建议。

优先级：P0。

## 11. 账户

现状：账户信息在仪表盘和账户页都展示。

改进：

- 清楚区分总权益、Perp 权益、Spot 余额、可用保证金、可提取、保证金率。
- 展示每个平台数据来源：Hyperliquid、MT5 account_info、paper fallback。
- fallback 到 paper 账户时必须明显提示，避免误以为是真实账户。
- 增加账户快照历史入口，支持查看权益变化和数据源变化。
- 账户页可展示各平台 readiness 阻塞原因。

优先级：P1。

## 12. 风控

现状：风控页展示风险状态和风险事件。

改进：

- 风控状态按“全局、平台、品种、账户、执行模式”分组。
- 每条阻塞原因给出对应设置入口和最后更新时间。
- readiness 结果和风险事件建立关联，方便判断为什么不能下单。
- 高风险开关变更要展示影响范围和回滚路径。
- Agent 被阻止执行建议时，在风控页可看到阻塞详情。

优先级：P0。

## 13. 日志

现状：日志页已有系统日志和站内告警，表格布局已接近占满。

改进：

- 支持按级别、分类、关键词、时间范围、对冲组 ID、订单 ID 筛选。
- 日志详情抽屉展示完整上下文，避免表格截断后不可读。
- 告警支持批量确认和确认备注。
- 系统日志、告警、审计、Agent 建议可以按同一时间线查看。
- 复制错误上下文时自动包含时间、分类、trace 或关联 ID。

优先级：P1。

## 14. 设置

现状：设置页字段多，容易出现布局兼容问题。

改进：

- 设置页按策略参数、风控参数、品种映射、MT5 交易时段、实盘开关分区，保持自然滚动。
- 每个分区顶部展示“保存状态、最近修改人、最近修改时间”。
- 敏感开关独立放到危险区域，不和普通输入混排。
- 输入字段增加单位、范围、默认值、当前生效值和保存后是否立即生效。
- 保存前做变更摘要，尤其是实盘、风控、自动执行、paper-live 探针相关字段。
- 品种映射页增加校验结果：HL 合约、MT5 symbol、合约规格、最小手数、交易时段。
- Agent 配置后续放在独立分区，默认 observe 或 advise，不默认自动执行。

优先级：P0。

## 15. Agent 中心

这是新增规划页面。

改进目标：

- 展示 Agent 当前运行模式、最近诊断时间、建议数量和高风险待确认数量。
- 建议列表支持按风险、品种、平台、对冲组、状态筛选。
- 建议详情展示证据、推理摘要、风控结果、readiness 结果和推荐动作。
- 高风险动作必须通过确认抽屉，不允许在悬浮窗里直接执行。
- Agent 历史形成可复盘时间线，关联日志、审计、订单、成交和对冲组事件。

优先级：P2，依赖 Agent 后端设计落地。

## 16. 优先级建议

### P0

- 统一表格页滚动和分页策略。
- 链路监控阻塞原因和下一步建议。
- 对冲组异常状态和危险操作确认抽屉。
- 执行记录异常订单排查。
- 仓位单腿、残仓、孤儿仓位高亮。
- 风控/readiness 阻塞原因可解释。
- 设置页布局兼容和危险设置隔离。

### P1

- 全局数据新鲜度。
- 仪表盘健康摘要和权益拆分。
- 价差研究和资金费研究图表口径说明。
- 账户数据源/fallback 提示。
- 日志详情和筛选增强。

### P2

- 登录页细节。
- 报价时差解释能力。
- Agent 中心和悬浮助手。
- 导出、复制诊断和跨页面时间线。
